## Project Overview

KACOS is a minimal Linux-based operating system for AI agents. It boots a Linux kernel (Alpine LTS) with a custom initramfs containing a lightweight init program that runs as PID 1.

## Build System

The top-level `Makefile` orchestrates the build:

**Prerequisites:**
- `qemu` (for testing)
- `x86_64-elf-gcc` (install via `brew install x86_64-elf-gcc`)
- `cpio` (usually preinstalled)

**Common commands:**

```bash
# Build initramfs and run in QEMU serial console
make run

# Run in QEMU with GUI window
make run-gui

# Clean build artifacts
make clean
```

**Manual build:**
```bash
x86_64-elf-gcc -nostdlib -static -O2 -o /tmp/kacos-init init.c
mkdir -p initramfs/{dev,proc,sys,tmp,bin}
cp /tmp/kacos-init initramfs/init
chmod +x initramfs/init
cd initramfs && find . | cpio -o -H newc | gzip > ../initramfs.cpio.gz
qemu-system-x86_64 -kernel vmlinuz -initrd initramfs.cpio.gz \
  -m 512M -nographic -append "console=ttyS0 panic=1"
```

## Architecture

### Boot Sequence

1. Linux kernel boots from `vmlinuz` (Alpine LTS pre-built kernel)
2. Kernel extracts and mounts `initramfs.cpio.gz`
3. Kernel executes `/init` as PID 1
4. `init.c` opens `/dev/console`, mounts proc/sys/devtmpfs/tmpfs
5. Interactive agent shell starts

### Init Program (`init.c`)

The init program uses raw Linux syscalls (no libc). Key design:
- `_start()` is the entry point — no libc startup code
- Syscalls use inline assembly with `syscall` instruction
- `sys_open("/dev/console")` + `dup2` to set up stdio before any output
- `shell_loop()` reads commands and dispatches

### Files

- `vmlinuz` — Pre-built Alpine Linux LTS kernel (do not modify)
- `initramfs.cpio.gz` — Gzipped cpio archive, generated by Makefile
- `init.c` — Custom init program, compiled with x86_64-elf-gcc
- `agent/` — Experimental Rust agent (not currently used in boot)

## Important Implementation Notes

- The init program must open `/dev/console` and `dup2` it to fds 0,1,2 before using any I/O. In initramfs, PID 1 starts with no open file descriptors.
- `x86_64-elf-gcc` is a bare-metal cross-compiler. It cannot link musl/glibc binaries correctly. The init program avoids libc entirely and uses raw syscalls.
- The Alpine kernel in `vmlinuz` is pre-built and pre-modified. Do not attempt to build a Linux kernel on macOS — it requires a Linux cross-compilation toolchain.
